# Config Options

The core configuration for a frame is defined in a JSON format. Below is an example configuration for a frame:

```json
{
    "menu": [
        // Menu items configuration
    ],
    "config": {
        // Color configuration
    },
    "display_edit_profile": false,
    "display_change_password": false
}
```

### Menu Configuration

The `menu` section defines the structure of the frame's menu. It consists of an array of objects, each representing a menu item. Each menu item can have the following properties:

- **url**: The URL for the menu item.
- **icon**: The base64-encoded SVG image representing the menu item icon.
- **name**: The name or label for the menu item.
- **children**: An optional array of nested menu items, forming a hierarchical structure.

### Color Configuration

The `config` section defines the color scheme for the frame. It includes properties such as `accent`, `header`, `primary`, `sidebar`, `secondary`, `background`, `typography`, and `headerBorder`. Developers can customize these values based on the desired color scheme for the application.

### Additional Configuration

- **display_edit_profile**: Boolean value indicating whether to display the "Edit Profile" option.
- **display_change_password**: Boolean value indicating whether to display the "Change Password" option.
